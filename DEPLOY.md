# دليل النشر — مجموعة أبو باشا

## الملفات المطلوب رفعها

ارفع **كل** محتويات المجلد (مسارات نسبية — بدون تغيير):

```
index.html
admin.html
data.js
storage.js
app.js
admin.js
style.css
antigravity.css
admin.css
assets/
  └── tarboosh.png
README.md          (اختياري)
DEPLOY.md          (اختياري)
.gitignore         (اختياري — للمستودع فقط)
```

لا ترفع ملفات `.env` أو نسخ احتياطية محلية.

---

## Netlify

1. سجّل في [netlify.com](https://www.netlify.com).
2. **Sites → Add new site → Deploy manually** (أو اربط GitHub).
3. اسحب مجلد المشروع كاملاً إلى منطقة الرفع.
4. **Publish directory:** الجذر `/` (لا يوجد build).
5. بعد النشر: `https://your-site.netlify.app/index.html`  
   لوحة التحكم: `https://your-site.netlify.app/admin.html`

### (اختياري) `_redirects` للروابط النظيفة

أنشئ ملف `_redirects` في الجذر:

```
/admin    /admin.html   200
```

## GitHub Pages

1. أنشئ مستودعاً وارفع الملفات.
2. **Settings → Pages → Source:** Deploy from branch `main`, folder `/ (root)`.
3. الموقع: `https://username.github.io/repo-name/`
4. لوحة التحكم: `.../admin.html`

> إذا كان الموقع في مسار فرعي (`/repo-name/`)، الروابط النسبية الحالية تعمل تلقائياً.

## cPanel / استضافة مشتركة

1. **File Manager → public_html** (أو مجلد الدومين).
2. ارفع كل الملفات مع الحفاظ على مجلد `assets/`.
3. تأكد أن `index.html` في الجذر.
4. افتح `https://yourdomain.com` — يجب أن تظهر الصفحة الرئيسية.
5. لوحة التحكم: `https://yourdomain.com/admin.html`

### SSL

فعّل **Let's Encrypt** من cPanel → SSL/TLS.

## الوصول إلى لوحة التحكم

| البند | التفاصيل |
|-------|----------|
| الرابط | `https://yourdomain.com/admin.html` |
| كلمة مرور | **لا يوجد** — الموقع ثابت بالكامل |
| التخزين | `localStorage` في متصفح من يفتح اللوحة |

### تأمين لوحة التحكم (موصى به)

- **cPanel:** Password Protect Directory على `admin.html` أو مجلد `/admin/`.
- **Netlify:** Basic Auth (خطة مدفوعة) أو لا ترفع `admin.html` للعامة واستخدمه محلياً فقط.
- **بديل:** احتفظ بـ `admin.html` على جهازك وانسخ البيانات يدوياً (غير عملي للصور).

> `admin.html` يحتوي `noindex, nofollow` لمنع فهرسته في محركات البحث.

## تحديث الأسعار اليومية

1. افتح `admin.html`.
2. **تبويب الأسعار والأصناف:**
   - عدّل **السعر الحالي** و**سعر أمس** لكل صنف.
   - الأسهم (▲▼) تُحسب تلقائياً.
3. في بداية كل يوم: اضغط **«نشر أسعار اليوم»** (ينقل الحالي → أمس) ثم عدّل الأسعار الجديدة.
4. اضغط **«حفظ كل التعديلات»** — إلزامي.
5. افتح `index.html` في تبويب جديد وتحقق من قسم **أسعار اليوم**.

### إضافة صنف جديد

**إضافة صنف** → املأ الاسم والسعر → **حفظ كل التعديلات**.

## رفع صور الهيرو والشعارات

1. `admin.html` → **صور الهيرو والشعارات**.
2. ارفع: شعار القائمة، صورة الهيرو، شعار الفوتر.
3. **حفظ كل التعديلات**.
4. حدّث الصفحة الرئيسية.

> الصور تُخزَّن كـ Base64 في `localStorage`. حجم كبير جداً قد يملأ التخزين (~5 MB حسب المتصفح).

## قائمة ما بعد النشر

- [ ] الصفحة الرئيسية تفتح بدون أخطاء (F12 → Console).
- [ ] قسم **أسعار اليوم** يعرض البطاقات مع الحركة (anti-gravity).
- [ ] روابط **واتساب** و**فيسبوك** و**tel:** تعمل.
- [ ] نموذج الطلب يفتح واتساب برسالة منسقة.
- [ ] القائمة الجوال (همبرger) تفتح وتغلق.
- [ ] الهيرو: «مجموعة» + «أبو باشا» + الطربوش يظهران بشكل صحيح على الجوال.
- [ ] استيراد JSON على الدومين بعد الرفع (إن وُجدت بيانات محلية).
- [ ] تعديل `data.js` إذا تغيّرت أرقام الهاتف قبل الرفع.
- [ ] تفعيل SSL (HTTPS).
- [ ] (اختياري) ربط الدومين المخصص في Netlify/cPanel.
- [ ] (اختياري) Google Search Console + تحديث `og:image` بعد رفع صورة الهيرو الحقيقية.

## قيود مهمة

| القيد | الشرح |
|-------|--------|
| **localStorage** | البيانات على **كل متصفح/جهاز** منفصلة. الزوار يرون البيانات الافتراضية من `data.js` ما لم تُصدَّر وتُشارك. |
| **لا خادم خلفي** | لا يوجد API — كل شيء في المتصفح. |
| **CDN فقط** | الخطوط (Google Fonts) والأيقونات (Font Awesome) من CDN — يحتاج اتصال إنترنت. |

### حل مشكلة «الزوار لا يرون تحديثاتي»

الموقع ثابت: تحديثات `localStorage` من `admin.html` **لا تصل للزوار** تلقائياً.

**الطريقة الموصى بها (تصدير/استيراد JSON):**

1. عدّل المحتوى في `admin.html` محلياً.
2. اضغط **«تصدير JSON»** واحفظ الملف.
3. بعد رفع الموقع، افتح `admin.html` على الدومين.
4. **«استيراد JSON»** ثم **«حفظ كل التعديلات»**.

**للزوار الجدد (بدون localStorage):** حدّث `initialProductsData` و`initialCategoriesData` في `data.js` قبل الرفع.
